
## ⚙️ MVP-архитектура: что делаем сразу, что откладываем

### 🔹 Этап 0 — “Живой скелет” (1–3 дня)

Цель: с нуля получить **хотя бы 1–5 роликов**, прошедших полный путь: взяли → обработали → залили.

Сразу делаем:

1. **Источник контента (Telegram + локальные файлы)**

   * Простейший Telethon-скрипт: скачать последние N медиа из заданных каналов в `/raw`.
2. **Мини-БД или CSV для учёта файлов**

   * Таблица `media_items` с id, source, path, status.
3. **Базовый FFmpeg-преобразователь**

   * Обрезка концов, scale до 9:16, overlay простой рамки/текста.
4. **Ручной/полуавтоматический залив**

   * На этапе MVP можно заливать через YouTube Studio руками, чтобы не упираться сразу в Selenium.

Откладываем:

* прокси,
* менеджмент десятков каналов,
* аналитику и A/B-тесты.

---

### 🔹 Этап 1 — “Маленький конвейер” (1–2 недели)

Цель: выйти на **стабильные 10–20 роликов/день** на 1–2 канала.

Делаем:

1. **Очереди и статусы (RAW→CLEAN→READY)**

   * Один скрипт “обработать всё RAW → CLEAN”; другой — “CLEAN → READY”.
2. **Мини-CTA-генератор**

   * Шаблон описания и закрепленного комментария с одной фиксированной ссылкой.
3. **Полуавто-загрузчик**

   * Selenium-скрипт, который открывает YouTube, заполняет форму и ждёт вашего “ок”/капчу при необходимости.
4. **Простейшая аналитика**

   * Скрипт/таблица, где вручную раз в несколько дней записываете просмотры основных роликов и клики по ссылке (из Telegram/CPA).

Откладываем:

* поддержку >2-3 платформ,
* сложные вариации уникализации (скорость/цвет/текст A/B),
* продвинутый менеджмент каналов.

---

### 🔹 Этап 2 — “Фарм-режим light” (3–6 недель)

Цель: **масштабировать до нескольких каналов и десятков роликов/день** без ручного адского труда.

Делаем:

1. **Менеджмент каналов (минимальный)**

   * Таблица аккаунтов, привязка к отдельным профилям браузера/куки.
2. **Простейший прокси-слой**

   * Поддержка хотя бы нескольких IP/прокси и привязка “канал → прокси”.
3. **Полноценный uploader**

   * Очередь `ready` → автоматический залив по расписанию (ночь/утро).
4. **Авто-логирование статистики**

   * Скрипт, который раз в день/два вытаскивает просмотры роликов, считает агрегаты по каналу.

Откладываем:

* сложные анти-детект решения,
* умный health-monitor каналов,
* full-scale A/B-тесты и ML-оптимизацию.

---

### 🔹 Этап 3 — “Настоящий фарм” (если предыдущие этапы вообще дали смысл)

Цель: если метрики хоть как-то радуют, **вывести в плюс за счёт масштаба**.

Делаем (только если есть реальные деньги/трафик):

1. Нормальную **прокси-схему** (много IP, антидетект-браузеры).
2. Расширенный **менеджмент каналов** (ротация, авто-замена умерших).
3. **Общие тулзы для генерации A/B-версий** роликов.
4. Более глубокую **аналитику по офферам** (доход/1000 просмотров, сравнение вертикалей).

Если на Этапе 2 всё дохлое — сюда можно вообще не идти.
