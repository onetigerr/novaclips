## План (сквозной пайплайн)

- Определение выпуска: тема → 1 обещание результата → 3–7 ключевых тезисов → 1 “протокол/что сделать сегодня”.
- Сценарий: hook (0–10с) → карта ролика (что будет дальше) → блоки тезисов → итог → CTA.
- Шот‑лист по сценам: “смысловой блок → какой визуальный тип” (персонаж/схема/пример/чек‑лист/таймлайн), а не “картинки по субтитрам”.
- Озвучка: генерите голос по финальному сценарию; аудио становится источником истины по таймингу.
- Видеоряд: генерите/подбираете визуалы под сцены, затем собираете видео (темп + переходы + легкий motion).
- Субтитры: делаете автосубтитры в CapCut из готового аудио, правите только критические места (термины/числа/имена), экспорт финала.


## Контрольные точки (gates)

- Gate A (до озвучки): сценарий проходит “проверку на ценность” — есть ли конкретные пункты/шаги/список, или это просто рассказ.
- Gate B (после озвучки): проверка темпа — нет ли длинных кусков без нового тезиса/переключателя сцены.
- Gate C (после сборки видео): проверка читаемости/усталости — визуалы меняют типы (схема ↔ пример ↔ список), а не просто “разные картинки”.
- Gate D (перед публикацией): политика/безопасность/дисклеймеры, чтобы контент выглядел оригинальным и не “массово произведённым/повторяющимся ради просмотров”.


## Do’s (что делать)

- Делайте формат “метод/шаги” или “каталог техник” — это универсальный удерживающий каркас для любой ниши.
- Держите аудио главным: тайминг сцен и субтитров подстраивайте под фактическую озвучку.
- Стандартизируйте визуальные блоки (3–6 типов) и чередуйте их каждые 10–30 секунд, чтобы удержание создавалось смыслом, а не “красивыми переходами”.
- Если делаете реалистично синтетические сцены/ситуации, отмечайте altered/synthetic через настройку в Studio; YouTube прямо требует disclosure для “meaningfully altered or synthetically generated content that seems realistic”.[^1]


## Don’ts (что не делать)

- Не клонируйте один и тот же шаблон выпуска “слегка перефразирован” — YouTube отдельно уточняет, что контент не должен быть mass-produced или repetitive и должен быть сделан для удовольствия/обучения зрителя, а не только ради просмотров.
- Не рассчитывайте, что “слоп‑картинки” всегда простят: иногда да, но для устойчивости канала вам нужен повторяемый авторский вклад (структура, выводы, примеры), иначе риск по монетизации/доверия выше.
- Не оставляйте авто‑субтитры без минимального QA, потому что ошибки в числах/терминах ломают доверие сильнее, чем слабый визуал.


## Артефакты (что хранить для автоматизации)

- 1‑страничник выпуска (обещание, тезисы, протокол, примеры).
- Сценарий vN (текст) + аудио vN (истина по времени) + шот‑лист (сцены).
- Пак визуальных шаблонов (типы сцен) + “паспорт персонажа” (если есть).
- Чек‑лист Gate A–D как definition of done.

Готово — это можно натягивать на любую нишу и потом детализировать уже отдельной LLM под ваши инструменты/модели.
