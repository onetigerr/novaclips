## 📘 Тех-описание модулей

### 1) Источники контента (Input Layer)

Модуль для автоматического получения сырого медиа (видео/гиф/картинки) из внешних источников.
Основные адаптеры:

* Telegram (Telethon/Pyrogram: `iter_messages`, фильтр по дате, типу медиа)
* YouTube/TikTok/и т.п. через `yt-dlp` или сторонние API/скрипты.
  Выход: файлы в `/storage/raw` + запись в БД (источник, дата, тип, длительность).

---

### 2) Контент-менеджмент (Media DB)

Лёгкая БД (SQLite/Postgres/CSV+пандас) с таблицей `media_items`.
Поля: id, путь к файлу, источник, дата, длительность, статус (RAW/CLEAN/READY/POSTED), хэш (MD5/перцептивный).
Задачи: исключать дубли, отслеживать, где ролик уже использован, и формировать очереди на обработку/загрузку.

---

### 3) Видеопреобразование / уникализация

Скрипт/сервис, который по данным из БД берёт RAW-файлы и прогоняет через FFmpeg.
Функции: обрезка концов, crop/scale до 9:16, изменение скорости/яркости, наложение рамок/текста, замена фонового звука.
Вход: список файлов + параметры шаблона; выход: новые файлы в `/storage/clean` + обновление статуса в БД.

---

### 4) Генерация финального ролика (Render)

Финальный шаг подготовки: упаковка уникализированного контента в формат, готовый к заливу.
Может собирать один ролик из одного клипа или из нескольких (компиляция).
Использует заранее заданные пресеты (FFmpeg фильтр-цепочки или CapCut шаблон) для единообразного визуального стиля.
Вывод: `/storage/ready` + метаданные (длина, разрешение, вариант A/B).

---

### 5) Менеджмент каналов (Accounts / Channels)

Модуль, который хранит и управляет наборами аккаунтов YouTube (и других платформ).
Данные: логины/куки/refresh-токены/ID каналов/текущий статус (OK/limited/strike/ban).
Задача: раздавать uploader’у “живой” канал, вести историю публикаций на каждом канале, отключать проблемные, при необходимости подменять на новые.

---

### 6) Прокси и окружение (Proxy / Identity Layer)

Слой, который отвечает за сетевую идентичность: прокси, User-Agent, fingerprint.
Привязывает: аккаунт → набор прокси → профиль браузера (Selenium/Playwright).
Задача — минимизировать корреляцию между каналами и снизить риск массовых банов по IP/отпечатку.

---

### 7) Автозалив / публикация (Uploader)

Сервис/скрипт, который открывает YouTube Studio (или мобильный веб) под нужным аккаунтом и загружает видео.
Инструменты: Selenium/Playwright, иногда API там, где это допустимо.
Функции: загрузка файла, задание заголовка/описания/тегов/видимости, привязка к плейлисту, логирование статуса (успех/ошибка/капча).

---

### 8) CTA и внешние ссылки (CTA/Link Builder)

Модуль, который генерирует описания и закреплённые комментарии с ссылками (Telegram, лендинг, CPA).
Использует шаблоны текста + UTM-метки для трекинга.
На вход: тип оффера/кампании, на выход: строка для описания + комментарий + хештеги.

---

### 9) Аналитика и трекинг (Analytics)

Собирает статистику по роликам и каналам: просмотры, лайки, клики по ссылкам (через UTM/редирект-сервис).
Источники: YouTube (API/парсинг), Telegram/лендинг-статистика, CPA-дешборды.
Цель: находить форматы/источники, которые дают лучший RPM (доход на 1000 просмотров/кликов), и под это масштабировать.

---

### 10) Оркестратор / Планировщик (Orchestrator)

Главный “дирижёр” пайплайна: планирует, когда что запускать, какой модуль на каких данных.
Инструменты: cron, Celery/RQ, простые очереди задач.
Решает: какие источники сегодня парсить, какие файлы отправить на обработку, какие ролики в каких каналах заливать, и когда проверять статистику.
